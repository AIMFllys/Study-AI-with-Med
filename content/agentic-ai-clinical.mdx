## 从"问答机器"到"临床助手"

大语言模型（LLM）虽然具备强大的医学知识，但在真实的临床环境中，医生不仅需要"背出知识"，还需要执行一系列复杂的流程：

1. 查阅电子病历（EHR）
2. 开具化验单/影像检查
3. 解读化验结果并结合病历史
4. 综合判断并给出处方
5. 预约随访随检

**Agentic AI（自主智能体）** 的核心突破在于赋予大模型**工具使用能力 (Tool Use)** 和**长程任务规划能力 (Planning)**。

<Callout type="tip">
  医学智能体不再是单向的“一问一答”，而是具备 `Thought -> Action -> Observation -> Final Answer` 闭环的数字医生（例如基于 ReAct 框架）。
</Callout>

## 真实世界实验：构建一个简单的 Agent

我们要实现的不再是一个简单的对话模型，而是一个能调用 `check_lab_results()` 的智能体演示。

<PythonRunner>
class MedicalAgent:
    def __init__(self):
        # 模拟可用的外部工具
        self.lab_db = {"potassium": 5.8, "sodium": 138, "creatinine": 1.9}
        
    def check_lab_results(self, test_name):
        print(f"[Action 工具调用]: 查询 {test_name} 结果...")
        return self.lab_db.get(test_name, "未查到结果")
        
    def run(self, prompt):
        print(f"👨‍⚕️ 医生指令: {prompt}\n")
        
        # 思考与规划
        print("🤖 [Thought]: 医生问患者血钾情况，我需要调用实验室工具。")
        val = self.check_lab_results("potassium")
        print(f"🔬 [Observation]: 血钾={val} mmol/L")
        
        # 二次推理
        print("🤖 [Thought]: 血钾5.8偏高（>5.5），可能有高钾血症风险，需立刻提醒。")
        
        # 最终回答
        print("\n✅ [Final Answer]: 患者最新血钾值为 5.8 mmol/L，属于高钾状态，建议结合心电图复查，并注意避免使用引发高钾的药物。")

agent = MedicalAgent()
agent.run("帮我看看33床张大妈的血钾危不危险？")
</PythonRunner>

## 结语

从辅助诊断工具到能够自主规划流转的 Agentic AI，这不仅是 AI 获取了“手脚”，更是整个临床 IT 架构从“信息化”向“自动化中枢”的演变。未来的医院，将是由超级医生带领的一群 24 小时不知疲倦的数字医学 Agent 协同作战的网络。
